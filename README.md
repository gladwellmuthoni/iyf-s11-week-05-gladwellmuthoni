## Project Description
A dynamic, feature-rich To-Do List application built to master DOM manipulation and state management in JavaScript. This project demonstrates how to create, read, update, and delete (CRUD) tasks, filter lists by completion status, and manage application state efficiently without a framework.

## Technologies Used
-HTML5 (Semantic Structure)
-CSS3 (Flexbox, Dynamic Styling)
-Vanilla JavaScript (ES6+)

## Features
- Add Tasks: Create new tasks via input form.
-Toggle Completion: Click a task to mark it as complete/incomplete (visual strikethrough).
-Delete Tasks: Remove individual tasks.
-Filter Tasks: Toggle between "All", "Active", and "Completed" views.
-Stats Tracking: Real-time count of remaining items.
-Clear Completed: Batch remove all finished tasks.

## How to Run
1. Clone this repository
2. Open `index.html` in your browser

## Lessons Learned
-DOM Manipulation: Gained confidence in selecting, creating, and removing elements dynamically.
-Event Delegation: Learned how to optimize performance by attaching a single event listener to a parent element rather than individual children.
-State Management: Understood the importance of keeping the UI in sync with a central data array (todos) rather than manipulating the DOM directly.
-Event Handling: Mastered the use of the Event Object to capture user inputs and keyboard shortcuts

## Challenges Faced
Challenge: Managing the "active" state of filter buttons.
Solution: Added a logic check to remove the .active class from all buttons before applying it to the clicked one, ensuring only one filter is highlighted at a time.
